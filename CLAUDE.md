## Commands

```bash
npm run dev          # Start local development server
npm run build        # Production build (Next.js build + sitemap generation)
npm run start        # Start production server
npm run check-links  # Validate all internal links (starts dev server, checks, kills it)
make latest-docs COMMIT=<SHA>  # Sync docs from upstream wavs-foundry-template
```

## Architecture

This is a **Next.js 14 + Fumadocs** documentation site for WAVS (WASI AVS Runtime), deployed to Vercel at https://docs.wavs.xyz/.

### Content vs. Code separation
- **Content** lives in `/content/docs/` as MDX files. Much of it is synced from the upstream `wavs-foundry-template` repo via `scripts/sync_latest_docs.py` and two GitHub Actions workflows (`.github/workflows/`).
- **Frontend/styling** lives here. The source of truth for tutorial/handbook content is upstream.

### Documentation structure
Navigation order is controlled by `meta.json` files in each section directory — not alphabetical order. The top-level structure is defined in `content/docs/meta.json`.

### Key framework pieces
- **Fumadocs** handles MDX processing, sidebar navigation, and search
- **CodeHike** provides advanced code blocks with annotations and scroll-coding (`components/scrollycoding.tsx`)
- **`app/source.ts`** wires Fumadocs to the content directory
- **`app/layout.config.tsx`** controls top-level navigation, branding, and sidebar behavior

### LLM/AI accessibility
The site exposes documentation for AI consumption via:
- `*.md` extension on any page URL (rewritten in `next.config.mjs`)
- `/llms.txt` and `/llms-full.txt` endpoints
- `/public/wavs-foundry-template.md` and `/public/wavs-wasi-utils.md` (auto-generated by the `generate-ai-docs` workflow using `repomix` and `rustdoc-md`)

### Styling
- Tailwind CSS with a custom very-dark theme (`0 0% 2%` background) defined in `tailwind.config.js`
- shadcn/ui components (new-york style) in `components/ui/`
- CodeHike theme: `min-dark`
